Подсистема автоматизированного тегирования и категоризации рассматривается как автономный компонент, который должен присваивать документу одну основную категорию и набор тегов, пригодных для навигации и дальнейшей интеграции через HTTP API. В требованиях отдельно отмечены работа с разнородным пользовательским контентом, единый pipeline для категории и тегов, интерпретируемый результат, поддержка коротких заметок и длинных документов, а также локальная обработка без обязательной зависимости от внешнего API.

В документе сравниваются варианты архитектуры и выбирается отдельный микросервис с HTTP API как наиболее подходящий формат для самостоятельной подсистемы. Язык реализации — Python, потому что для задачи критична зрелая экосистема NLP/ML. В качестве backend framework предпочтение отдаётся FastAPI благодаря типизации, схемам данных и автодокументации. Для NLP/ML‑стека выбирается комбинация `sentence-transformers`, `KeyBERT` и `scikit-learn`, а внешний LLM допускается только как опциональный слой постобработки, а не как основной механизм.

Отдельный раздел посвящён инфраструктуре. Для хранения документов, задач обработки и результатов предлагается PostgreSQL, а в список вспомогательных зависимостей входят Uvicorn, Pydantic, SQLAlchemy и Docker. Полноценная очередь сообщений и отдельный inference‑сервис сознательно не включаются в базовую реализацию, чтобы не усложнять подсистему раньше времени. В результате получается автономный технологический стек для сервиса категоризации и тегирования с локальными embeddings, формализованными API‑контрактами и воспроизводимым развёртыванием.
